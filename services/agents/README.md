# InfraAI Agents

Python agent helpers imported by the FastAPI backend.

This package contains:

- open-ended infrastructure intent classification
- guardrails and non-goals
- shared tool handlers for chat and MCP
- evidence-query and score-driver explanation tools
- tool-based agent response generation
- agent review helpers for report reliability
- optional OpenAI explanation adapter

The first production rule is that agents explain structured model outputs. They
must not invent readiness scores, approvals, permits, funding decisions, grid
capacity guarantees, or real-world feasibility claims.

The deterministic scorer remains the source of truth. The LLM layer can only
rewrite and explain computed tool outputs, an existing `SiteAnalysisResult`, or
the report-review fallback generated by the Python tools.

Agents can answer general platform questions without a selected site. Site-
specific planning questions still require a selected location or current
analysis so responses can be grounded in evidence, score drivers, data gaps, and
guardrails.
